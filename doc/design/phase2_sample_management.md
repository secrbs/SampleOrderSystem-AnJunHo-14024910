# Phase 2: 시료 관리

## 목표

시스템의 기본 단위인 시료(Sample)의 등록, 목록 조회, 검색 기능을 구현한다.

## 도메인 모델

```cpp
struct Sample {
    std::string id;              // 예: "S-001"
    std::string name;            // 예: "실리콘 웨이퍼-8인치"
    double avgProductionTime;    // min/ea (예: 0.5)
    double yield;                // 수율 0.0 < yield <= 1.0 (예: 0.92)
    int stock;                   // 현재 재고 수량
};
```

> `Sample` 모델과 `SampleRepository`는 Phase 1에서 구현 완료.

## 구현 범위

### 시료 등록
- 입력: ID, 이름, 평균 생산시간(min/ea), 수율(0~1)
- 중복 ID 등록 불가
- 수율 범위 검증 (0.0 초과 ~ 1.0 이하)
- 등록 성공 시 JSON 파일에 즉시 저장

### 시료 목록 조회
- 등록된 전체 시료를 테이블로 출력
- 표시 항목: ID, 시료명, 평균 생산시간, 수율, 현재 재고

### 시료 검색
- 이름 부분 일치(대소문자 무시) 검색
- 결과 없으면 안내 메시지 출력

### SampleView 공통 UI 규칙
- 모든 서브 화면 상단에 내비게이션 경로 표시 (CLAUDE.md UI 규칙 참조)
- 사용자가 어떤 입력으로 현재 화면에 도달했는지 표시
