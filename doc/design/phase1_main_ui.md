## UI 요구사항

### 실시간 시각
- 메인 화면 우상단에 현재 시각 표시
- 입력 대기 중에도 1초마다 자동 갱신 (커서 위치 이동으로 해당 부분만 재출력)
- 화면 전체를 다시 그리지 않음

### 화면 구분
- 상태 영역: 시스템 현황 (시료 수, 재고, 주문 수, 생산 대기)
- 메뉴 영역: 기능 목록
- 입력 영역: 선택 프롬프트 (선택 > _) — 하단에 고정, 시각적으로 분리

### 입력 처리
- _getch() 사용 — Enter 불필요, 키 하나로 즉시 처리
- 입력 대기 중 _kbhit() 루프로 시각 갱신

## MVC 책임

| 레이어 | 책임 |
|--------|------|
| MainController | SystemStatus 계산 (시료 수, 총 재고, 주문 수, 생산 대기) |
| MainView | 화면 렌더링, 실시간 시각 갱신, 키 입력 수신 |

- 시각(time)은 View 내부에서 계산 (순수 표시 책임)
- SystemStatus 수치는 Controller → View로 전달

## 구현 범위

- model/SystemStatus.h — 시스템 현황 구조체
- controller/MainController — SystemStatus 집계
- view/MainView — 화면 렌더링, 실시간 시각, 입력
- repository/SampleRepository, OrderRepository — 파일 로드/저장 뼈대
- main.cpp — 진입점
